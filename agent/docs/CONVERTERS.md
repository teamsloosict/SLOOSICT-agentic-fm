# Script Format Converters

This document catalogues every converter in the agentic-fm project — tools that transform FileMaker script representations between formats. Knowing which tool handles which conversion prevents duplication and ensures the right tool is used for each job.

---

## Formats

| Format | Description | Where found |
|--------|-------------|-------------|
| **SaXML** | FileMaker's "Save a Copy as XML" export. Verbose, nested `<ParameterValues>` structure. Read-only — never generated by agentic-fm. | `agent/xml_parsed/scripts/{solution}/` |
| **fmxmlsnippet** | FileMaker's clipboard format. Flat `<Step>` elements with direct child elements. This is what gets pasted into Script Workspace. | `agent/sandbox/`, clipboard |
| **HR** | Human-readable script text matching Script Workspace display. Tab/space indented, one line per step, `//` prefix for disabled steps. | `agent/xml_parsed/scripts_sanitized/`, diff payloads, terminal output |

---

## Conversion Matrix

```
                    ┌──────────┐
      saxmlpreview  │          │  fm_xml_to_snippet
    SaXML ─────────►│    HR    │◄──────── fmxmlsnippet
      .py           │          │  snippet_to_hr.py
                    └──────────┘
                         ▲ ▲
         exploder        │ │        xml-to-hr.ts
      (Rust, build-time) │ │     (browser, client-side)
                         │ │
                    SaXML  fmxmlsnippet

                    ┌──────────┐
                    │          │
      SaXML ───────►│ snippet  │◄──────── HR
    fm_xml_to_      │          │  hr-to-xml.ts
    snippet.py      └──────────┘  (browser, client-side)
```

| From | To | Tool | Location | Runtime |
|------|----|------|----------|---------|
| SaXML → fmxmlsnippet | `fm_xml_to_snippet.py` | `agent/scripts/` | Python (server) |
| SaXML → HR | `saxmlpreview.py` | `.claude/skills/script-preview/scripts/` | Python (server) |
| SaXML → HR | fm-xml-export-exploder | External Rust tool | Build-time only |
| fmxmlsnippet → HR | `snippet_to_hr.py` | `agent/scripts/` | Python (server) |
| fmxmlsnippet → HR | `xml-to-hr.ts` | `webviewer/src/converter/` | Browser (client) |
| HR → fmxmlsnippet | `hr-to-xml.ts` | `webviewer/src/converter/` | Browser (client) |

### Conversions that do NOT exist (and why)

| Conversion | Why not needed |
|-----------|----------------|
| fmxmlsnippet → SaXML | SaXML is input-only (from FM exports). No tool consumes it downstream. |
| HR → SaXML | Same — SaXML is never an output target. |

## Server-Side Tools (Python)

### `agent/scripts/fm_xml_to_snippet.py`

**SaXML → fmxmlsnippet**

Converts a FileMaker "Save As XML" script file into fmxmlsnippet clipboard format. This is the bridge between what FileMaker exports and what gets pasted back in.

```bash
python3 agent/scripts/fm_xml_to_snippet.py \
  "agent/xml_parsed/scripts/{solution}/{path}.xml" \
  "agent/sandbox/OutputName.xml"
```

- 48 hand-coded translators for high-priority steps (control flow, Set Variable, Perform Script, etc.)
- Catalog-driven generic fallback handles all remaining steps automatically
- 100% coverage across all step types found in the codebase (128 unique types across 2,325 scripts)
- Maps nested SaXML `<ParameterValues>` to flat fmxmlsnippet child elements
- See `agent/scripts/XML_TRANSFORMATION.md` for the structural mapping reference

### `agent/scripts/snippet_to_hr.py`

**fmxmlsnippet → HR**

Converts fmxmlsnippet XML to human-readable script text. This is the server-side equivalent of the webviewer's `xml-to-hr.ts` — used for generating diff payloads and terminal previews.

```bash
# With line numbers (default)
python3 agent/scripts/snippet_to_hr.py agent/sandbox/MyScript.xml

# Raw text without line numbers (for diffs)
python3 agent/scripts/snippet_to_hr.py agent/sandbox/MyScript.xml --raw

# Write to file
python3 agent/scripts/snippet_to_hr.py agent/sandbox/MyScript.xml --raw -o output.txt
```

- Hand-coded renderers for 21 structurally unique steps (control flow, Set Variable, Perform Script, Set Field, etc.)
- Catalog-driven fallback for all other steps via `agent/catalogs/step-catalog-en.json`
- Handles indentation, disabled step prefixing, multi-line calculations
- Importable as a module: `from agent.scripts.snippet_to_hr import snippet_to_hr`

### `.claude/skills/script-preview/scripts/saxmlpreview.py`

**SaXML → HR**

Converts SaXML (not fmxmlsnippet) to Script Workspace format. Used by the `script-preview` skill to display existing scripts before modification.

```bash
python3 .claude/skills/script-preview/scripts/saxmlpreview.py \
  "agent/xml_parsed/scripts/{solution}/{path}.xml"
```

- Reads from the SaXML `<ParameterValues>` structure (different from fmxmlsnippet)
- Numbered output (tab-separated) for terminal display
- Not suitable for fmxmlsnippet input — use `snippet_to_hr.py` instead

## Client-Side Converters (TypeScript — Webviewer)

All located in `webviewer/src/converter/`.

### `xml-to-hr.ts`

**fmxmlsnippet → HR** (browser-side)

The webviewer's built-in converter for displaying script content in the editor. Runs in the browser via DOMParser.

- Entry point: `xmlToHr(xml: string): string`
- Delegates to step-specific converters registered in `step-registry.ts`
- Same indentation/disable logic as the Python converters

### `hr-to-xml.ts`

**HR → fmxmlsnippet** (browser-side)

The webviewer's primary feature — converts human-readable script text into paste-ready fmxmlsnippet XML.

- Entry point: `hrToXml(text: string, resolver: IdResolver): ConversionResult`
- Parses HR via `parser.ts`, dispatches to step converters or catalog-driven fallback
- Resolves field/layout/script IDs via `id-resolver.ts` (queries CONTEXT.json)

### `catalog-converter.ts`

Generic converter for steps without hand-coded handlers. Maps HR parameters to XML elements using catalog `params[]` definitions. Handles types: boolean, enum, calculation, namedCalc, text, field, layout, script, flagElement.

### `step-registry.ts`

Registration system for both conversion directions:
- `registerHrToXml(converter)` / `registerXmlToHr(converter)`
- `getHrToXmlConverter(name)` / `getXmlToHrConverter(name)`
- XML helpers: `escXml()`, `cdata()`, `stepOpen()`, `stepSelfClose()`

### `steps/` — Per-Step Converters

| File | Steps handled |
|------|--------------|
| `control.ts` | `# (comment)`, If, Else If, Else, End If, Loop, Exit Loop If, End Loop, Exit Script, Set Variable, Allow User Abort, Set Error Capture, Perform Script, Halt Script |
| `fields.ts` | Set Field |
| `navigation.ts` | Go to Layout, Go to Object |
| `records.ts` | Commit Records/Requests, New Record/Request |
| `windows.ts` | Freeze Window, New Window, Close Window |
| `miscellaneous.ts` | Show Custom Dialog |

### `parser.ts`

HR line parser — splits script text into `ParsedLine` objects with step name, parameters, disabled state, and indentation. Handles multi-line bracket merging and nested quote/paren/bracket tracking.

## Build-Time Tool

### fm-xml-export-exploder (Rust)

**FileMaker XML export → SaXML + HR**

External Rust tool that parses a monolithic FileMaker "Save As XML" export into per-entity files. Generates both `scripts/` (SaXML) and `scripts_sanitized/` (HR) output. Not part of this repository — lives in `external_tools/fm-xml-export-exploder/`.

## Diff Workflow

The standard workflow for showing before/after diffs of refactored scripts:

```bash
# 1. "Before" HR — read from pre-generated scripts_sanitized
BEFORE=$(cat "agent/xml_parsed/scripts_sanitized/{solution}/{path}.txt")

# 2. "After" HR — convert refactored fmxmlsnippet to HR
AFTER=$(python3 agent/scripts/snippet_to_hr.py agent/sandbox/Refactored.xml --raw)

# 3. Push to webviewer (Monaco diff editor)
python3 -c "
import json, sys
payload = json.dumps({
    'type': 'diff',
    'before': sys.argv[1],
    'content': sys.argv[2],
    'repo_path': '/path/to/repo'
})
sys.stdout.write(payload)
" "$BEFORE" "$AFTER" | curl -s -X POST -H 'Content-Type: application/json' \
  -d @- http://host.docker.internal:8765/webviewer/push
```

The webviewer renders this as a side-by-side Monaco diff editor. The companion server writes the payload to `agent/config/.agent-output.json`, which the webviewer polls via `/api/agent-output`.

### Webviewer Diff Payload Schema

```json
{
  "type": "diff",
  "before": "<original HR text>",
  "content": "<modified HR text>",
  "repo_path": "/absolute/path/to/repo"
}
```

Other supported types: `preview` (read-only editor), `result` (read-only), `diagram` (Mermaid), `layout-preview` (HTML+CSS).

## Step Catalog

`agent/catalogs/step-catalog-en.json` is the shared reference used by all converters. Key fields:

| Field | Used by | Purpose |
|-------|---------|---------|
| `name` | All converters | Step identification |
| `id` | `saxmlpreview.py`, `fm_xml_to_snippet.py` | SaXML step dispatch |
| `hrSignature` | `saxmlpreview.py`, `snippet_to_hr.py` | HR parameter format |
| `params[]` | All converters | Parameter type → XML/HR mapping |
| `selfClosing` | XML generators | `<Step ... />` vs `<Step>...</Step>` |
| `blockPair` | Indentation logic | If/Loop open/middle/close roles |

See `agent/docs/SCHEMA_GUIDANCE.md` for the complete param type → XML mapping reference.
